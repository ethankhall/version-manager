Why Crom?

You may be wondering to yourself, why Crom? Well for most users, this is probably totally overkill. Crom is a service that gives you (a library/service/person trapped in a computer) the ability to query what versions of have been published. This becomes useful, when you want to be able to report on what's available.
One of Crom's goals it to remove the need to have bump commits. We'll discuss this below.

What Questions can YOU ask?

Crom aims to answer a few questions (and more will be added latter) without needing access to source repo. They are:
  • What versions are available of this repo?
  • What is the latest version for a repo?
  • Where is your closes release from your HEAD?
  • How do I product reports to be consumed by users (deployment reports, release notes, etc)

Version Management

One of the hard questions when dealing with a project is how do you want to version. Crom provides a way to provide the versions to a project. It does this by reading your commit message and searching for keywords like [bump major]. If Crom see's that string it will take the last commit in your graph and bumps the major version for you.
You may ask, why put that in a service?! Well because then you don't need to manage a file who's only purpose is to record the version number and needs to update with ever version.
Keyword Meaning
[bump major] Bumps the Major Version of the repo
[bump minor] Bumps the Minor Version of the repo
[bump patch] Bumps the Patch Version of the repo
[bump build] Bumps the Build Version of the repo
[bump group n] - Usage: [bump group 6] Bumps the n'th part of the version number. This is base 0, meaning that 0 is the major version.
[force version {{ '{' }}versionNumber{{ '}' }}] - Usage: [force version 1.2.183.24] Sets the version to be versionNumber.
